# StickerBoard — Design Book (proposal, for approval)

This folder is a **design exploration**, not product code. Nothing here is wired
into the app. Its job is to let Pasha *see and approve* UI elements before any
implementation (per CLAUDE.md working principle #4).

## How to view

Open `design-book/index.html` in any browser (double-click it). Use the three
buttons at the top to switch between variants. Hover/click elements to feel the
motion prototypes (sticker drop, button press, pin pulse, save confetti).

## The three directions

1. **Atelier** — refined / editorial / minimal. Warm paper-white, muted ink,
   one confident accent. "Looks simple, works brilliantly." The grown-up option.
2. **Confetti** — cute / playful / tactile. Pastels, rounded everything, tape on
   stickers, bouncy motion, confetti on save. The joyful option.
3. **Midnight** — premium / dark / focused. Glassy dark surfaces, neon accents,
   subtle glow. The "pro tool" option.

## What to review

For each variant, judge: color + type tokens, buttons, sticky-note sticker,
map pin (public vs private), board card, bottom toolbar, visibility toggle,
share modal, and the two screen mockups (Map, Board editor).

Tell me, per element: **approve / change / mix-and-match across variants.**
Approved elements become the real design tokens + components.
